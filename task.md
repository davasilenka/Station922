# Реализация поддержки CGI с асинхронным вводом-выводом через pipes

Необходимо добавить в веб-сервер Station922 поддержку CGI (Common Gateway Interface) для выполнения серверных скриптов. Сервер должен поддерживать как стандартный CGI, так и передачу данных через асинхронные pipes с использованием Windows API.

## Требования к функциональности

**Базовая поддержка CGI:**

- Выполнение CGI-скриптов по расширениям файлов (.cgi, .exe, .pl, .py, .php и другие, настраиваемые в конфигурации)
- Установка всех необходимых переменных окружения CGI (REQUEST_METHOD, QUERY_STRING, CONTENT_TYPE, CONTENT_LENGTH, SCRIPT_NAME, PATH_INFO, PATH_TRANSLATED, REMOTE_ADDR, SERVER_NAME, SERVER_PORT, HTTP_* заголовки)
- Поддержка передачи тела запроса (POST/PUT данных) в stdin процесса CGI
- Получение ответа от CGI-скрипта и передача его клиенту


**Асинхронный ввод-вывод:**

- Использование асинхронных pipes для передачи данных в stdin и из stdout/stderr CGI-процесса
- Интеграция с существующей системой async I/O сервера (IAsyncIoTask, IBaseAsyncStream)
- Использование CreateProcess с перенаправлением stdin/stdout/stderr через pipes
- Асинхронное чтение вывода CGI-скрипта без блокировки рабочих потоков


**Обработка ответа CGI:**

- Парсинг заголовков ответа CGI (формат "Header: Value")
- Обработка Status-заголовка для установки кода ответа HTTP
- Обработка Content-Type и других стандартных заголовков
- Поддержка NPH (Non-Parsed Headers) CGI-скриптов
- Передача тела ответа клиенту с использованием существующих механизмов async записи


**Управление процессами:**

- Установка таймаута на выполнение CGI-скрипта (настраиваемый в конфигурации)
- Принудительное завершение процесса при превышении таймаута
- Обработка завершения процесса (нормальное, по ошибке, по таймауту)
- Ограничение размера вывода CGI-скрипта для защиты от DoS


**Безопасность и конфигурация:**

- Настройка директорий, в которых разрешено выполнение CGI-скриптов
- Возможность указания интерпретатора для различных расширений файлов
- Настройка таймаута выполнения CGI-скрипта
- Настройка максимального размера входных данных
- Настройка максимального размера выходных данных
- Поддержка chroot/jail для CGI-процессов (опционально)


**Обработка ошибок:**

- Возврат ошибки 500 при невозможности запуска CGI-процесса
- Возврат ошибки 502 (Bad Gateway) при ошибке CGI-скрипта
- Возврат ошибки 504 (Gateway Timeout) при превышении таймаута
- Логирование ошибок выполнения CGI
- Обработка некорректного ответа CGI (отсутствие пустой строки после заголовков и т.д.)


## Архитектурные требования

**Интеграция с существующей архитектурой:**

- Создание нового процессора HttpCgiProcessor (аналогично HttpGetProcessor, HttpPutProcessor)
- Реализация интерфейса IHttpAsyncProcessor
- Интеграция с HttpProcessorCollection
- Использование существующих механизмов IAsyncResult для асинхронных операций
- Использование пула потоков (ThreadPool) для асинхронных операций с pipes


**Структура кода:**

- Создание модуля CgiProcess.bas/bi для управления CGI-процессами
- Создание модуля CgiAsyncTask.bas/bi для асинхронных операций с pipes
- Создание модуля HttpCgiProcessor.bas/bi для обработки HTTP-запросов к CGI
- Расширение интерфейса IWebSite для поддержки CGI-конфигурации
- Расширение парсера INI-конфигурации для новых параметров CGI


**Асинхронное выполнение:**

- Неблокирующее чтение из stdout CGI-процесса
- Неблокирующая запись в stdin CGI-процесса (для POST/PUT данных)
- Корректная обработка ситуации, когда данные приходят частями
- Корректная очистка ресурсов (закрытие handles, завершение процесса) при ошибках


## Технические детали

**Windows API для использования:**

- CreateProcess с флагами для перенаправления stdio
- CreatePipe для создания pipes
- SetHandleInformation для наследования handles
- CreateIoCompletionPort для асинхронного ввода-вывода
- ReadFile/WriteFile с OVERLAPPED структурами
- GetExitCodeProcess для проверки завершения процесса
- TerminateProcess для принудительного завершения
- CreateEnvironmentBlock/SetEnvironmentVariable для переменных окружения


**Переменные окружения CGI (обязательные):**

- AUTH_TYPE, CONTENT_LENGTH, CONTENT_TYPE, GATEWAY_INTERFACE
- PATH_INFO, PATH_TRANSLATED, QUERY_STRING, REMOTE_ADDR
- REMOTE_HOST, REMOTE_IDENT, REMOTE_USER, REQUEST_METHOD
- SCRIPT_NAME, SERVER_NAME, SERVER_PORT, SERVER_PROTOCOL
- SERVER_SOFTWARE
- Все HTTP_* заголовки из запроса клиента


## Критерии приемки

**Функциональность:**

- Успешное выполнение CGI-скриптов на Python, Perl, PHP (через CGI-режим), cmd/bat
- Корректная передача переменных окружения и query string
- Корректная обработка POST-запросов с передачей тела в CGI
- Корректный возврат ответа от CGI клиенту
- Работа с асинхронным вводом-выводом без блокировки сервера


**Нагрузочное тестирование:**

- Сервер должен выдерживать одновременное выполнение нескольких CGI-скриптов
- Корректная обработка ситуации, когда CGI-скрипт "зависает" (таймаут)
- Корректная обработка ситуации, когда CGI-скрипт выводит очень много данных
- Отсутствие утечек handles и памяти при длительной работе


**Конфигурация:**

- Возможность включать/отключать CGI для отдельных сайтов
- Возможность указывать расширения файлов для CGI
- Возможность указывать интерпретаторы для различных типов скриптов
- Настройка таймаутов и лимитов
