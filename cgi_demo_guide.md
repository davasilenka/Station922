# Демонстрация CGI для преподавателя

## Подготовка

1. Запустить сервер: `Station922_x64.exe`
2. Открыть браузер (Chrome/Яндекс)
3. Для части демонстраций открыть DevTools: `F12` --> вкладка `Network`

---

## Демонстрация 1: Базовый запуск CGI

**URL:** `http://localhost/cgi-bin/hello.py`

**Что показывает:**
- Сервер выполнил Python-скрипт, а не просто отдал файл
- В ответе `Hello, CGI!` + все переменные окружения CGI

**Пункты task.md:** выполнение CGI-скриптов, установка переменных окружения

**На что обратить внимание:** строка `GATEWAY_INTERFACE=CGI/1.1` -- сервер реализует стандарт CGI

---

## Демонстрация 2: Запуск не-Python скрипта

**URL:** `http://localhost/cgi-bin/hello.bat`

**Что показывает:**
- Сервер выполняет `.bat`-файл через командную оболочку Windows
- Поддержка разных расширений, настройка интерпретаторов

**Пункты task.md:** выполнение CGI-скриптов по расширениям файлов, указание интерпретатора

**На что обратить внимание:** URL тот же домен `/cgi-bin/`, но расширение `.bat` вместо `.py`, ответ `Hello from BAT`

---

## Демонстрация 3: Query string

**URL:** `http://localhost/cgi-bin/env.py?name=Student&group=42`

**Что показывает:**
- В выводе `QUERY_STRING=name=Student&group=42` -- сервер передал параметры скрипту
- `REQUEST_METHOD=GET`

**Пункты task.md:** передача query string, переменные окружения CGI

---

## Демонстрация 4: Управление HTTP-статусом из скрипта

**URL:** `http://localhost/cgi-bin/status.py?404+Not+Found`

**Что показывает:**
- Браузер показывает ошибку 404
- Ошибку вернул НЕ сервер, а скрипт через заголовок `Status: 404 Not Found`

**Как проверить через DevTools:**
- `F12` --> `Network` --> обновить страницу --> кликнуть на запрос
- В `Response Headers` видно, что ответ сформирован CGI-скриптом

**Другие варианты:**
- `http://localhost/cgi-bin/status.py` --> 200 (по умолчанию)
- `http://localhost/cgi-bin/status.py?302+Found` --> редирект (браузер перейдёт)

**Пункты task.md:** обработка Status-заголовка, обработка ответа CGI

---

## Демонстрация 5: Таймаут (защита от зависания)

**URL:** `http://localhost/cgi-bin/slow.py`

**Что показывает:**
- Скрипт `slow.py` спит 60 секунд
- Через 3 секунды сервер возвращает **504 Gateway Timeout**
- Сервер принудительно завершил процесс скрипта

**Пункты task.md:** таймаут CGI, принудительное завершение, обработка ошибок

---

## Демонстрация 6: POST-запрос с телом

**Способ 1 -- PowerShell (открыть отдельное окно):**
```powershell
Invoke-WebRequest -Uri "http://localhost/cgi-bin/post_echo.py" -Method POST -Body "Hello, POST body!"
```
Ответ содержит `Hello, POST body!` -- скрипт прочитал тело запроса из stdin.

**Способ 2 -- curl (если установлен):**
```
curl -X POST -d "Hello, POST body!" http://localhost/cgi-bin/post_echo.py
```

**Пункты task.md:** передача тела запроса в stdin, поддержка POST/PUT

## Демонстрация 7: HTTP-заголовки клиента

**PowerShell:**
```powershell
Invoke-WebRequest -Uri "http://localhost/cgi-bin/env.py" -Headers @{"X-Custom-Header"="TestValue"}
```

**Что показывает:**
- В выводе `HTTP_X_CUSTOM_HEADER=TestValue` -- сервер пробросил заголовки как `HTTP_*` переменные

**Пункты task.md:** передача HTTP_* заголовков в CGI

## Демонстрация 8: Коды ошибок

**URL 1:** `http://localhost/cgi-bin/nonexistent.py`
- Возвращает **404** -- скрипт не найден

**URL 2:** `http://localhost/cgi-bin/exit1.py`
- Возвращает **502 Bad Gateway** -- скрипт завершился с ошибкой (exit code 1)

**URL 3:** (демонстрация 5) `slow.py`
- Возвращает **504 Gateway Timeout** -- превышение таймаута

**Пункты task.md:** обработка ошибок (404/502/504)

---

## Демонстрация 9: Заголовки ответа через DevTools

1. Открыть `F12` --> `Network`
2. Зайти на `http://localhost/cgi-bin/hello.py`
3. Кликнуть на запрос, вкладка `Headers` --> `Response Headers`

**Видно:**
- `Content-Type: text/plain` -- его установил СКРИПТ (print "Content-Type: text/plain")
- `Server: Station922/1.0` -- его добавил сервер
- Если скрипт возвращает `Status:`, он преобразуется в HTTP-статус

**Пункты task.md:** парсинг заголовков ответа CGI, обработка Content-Type

## Демонстрация 10: Большой вывод

**URL:** `http://localhost/cgi-bin/bigoutput.py?200000`

**Что показывает:**
- Скрипт генерирует 200 КБ данных
- Сервер успешно передаёт большой объём клиенту
- Лимит на вывод работает (в конфиге `CgiMaxOutputSize`)

**Пункты task.md:** ограничение размера вывода, передача тела ответа

## Демонстрация 11: Параллельные запросы

**PowerShell (открыть отдельное окно):**
```powershell
1..5 | ForEach-Object -Parallel {
    Invoke-WebRequest -Uri "http://localhost/cgi-bin/hello.py" -UseBasicParsing
} -ThrottleLimit 5
```

**Что показывает:**
- Все 5 запросов возвращают 200 одновременно
- Сервер обрабатывает несколько CGI-процессов параллельно

**Пункты task.md:** нагрузочное тестирование, одновременное выполнение CGI

*(Примечание: PowerShell 5.1 не поддерживает `-Parallel`, нужен pwsh.exe или замена.)*

**Альтернатива (проще):**
Просто открой `hello.py` в 5 вкладках браузера быстро -- все загрузятся нормально.

---

## Что НЕ демонстрировать (пока не готово)

- Асинхронный ввод-вывод через IOCP -- pipes работают синхронно (фаза 2)
- Stderr CGI-скриптов не читается -- ошибки не попадают в лог
- Нет ограничения по разрешённым директориям -- CGI можно запустить из любой папки
- При 9+ одновременных CGI сервер начнёт отказывать (закончились worker-потоки)
